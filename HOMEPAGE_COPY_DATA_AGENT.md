更新时间：2026-03-18

## 使用原则

这版首页文案遵循三个原则：

- 不把 Inkly Cat 说成万能 agent
- 不过早承诺特定学科深度
- 把重点放在真实可感知的数据工作闭环上

同时保留一层更有人味的品牌气质：

- 用研究的态度处理真实问题
- 保持认真、好奇和讲依据的感觉
- 允许产品既专业，也有一点生活里的反差感

建议首页整体语气保持：

- 简单
- 克制
- 具体
- 少形容词，多结果描述

## 第二部分：为什么会需要 Inkly Cat

### 标题

很多数据工作，不复杂，但很耗人

### 正文

真正耗时间的，往往不是某一步不会做，而是总要在表格、脚本、图表和结果整理之间来回切换。

文件要处理，脚本要运行，图表要调整，结果还要重新整理成能继续使用或交付的内容。

Inkly Cat 想做的事情很明确：把这些零散步骤接起来，让数据工作更顺一点。

## 品牌气质方向

这一部分不建议替代主定位，更适合作为首页辅助文案、品牌介绍、社区内容主题或宣传语。

建议关系如下：

- 主定位负责说明产品是做什么的
- 品牌气质负责说明产品相信什么、吸引什么样的人

### 推荐品牌句

用研究的态度，处理真实问题

### 备选品牌句

- 把问题认真研究一下
- 认真研究每一个问题
- 生活里的问题，也值得认真研究
- 用研究的方式，看待日常问题
- 用分析和依据，把事情说明白

### 为什么这个方向成立

很多用户并不只想“得到一个答案”，而是喜欢把问题拆开、看数据、找依据、做比较。

这种态度不只存在于正式研究里，也会自然延伸到日常判断：

- 最近的天气适不适合看樱花
- 这周的风况适不适合放风筝
- 哪种安排更省时间
- 哪个方案更值得尝试

这种“把小事也认真研究一下”的气质，会让 Inkly Cat 比传统工具更有记忆点。

### 适合放置的位置

- 首页 Hero 下方的小字
- About 页面
- 社交媒体简介
- 示例内容页标题
- 社区活动和内容栏目

## 对设计和截图的要求

截图里尽量满足这些条件：

- 任务标题具体，不要写“demo”或“example”
- 图表看起来已经接近真实工作结果
- 文案短，不要满屏文字
- 不要展示明显还没做好的区域
- 尽量避免空白态和占位态
- 截图之间保持同一套视觉风格
