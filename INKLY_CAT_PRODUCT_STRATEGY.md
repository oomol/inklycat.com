# Inkly Cat 产品定位与发展路径

更新时间：2026-03-18

## 一句话定位

Inkly Cat 不是另一个通用 AI agent，而是一个面向科研和高要求分析场景的结果型 agent。

它的核心价值是：

> 从原始数据到可发表图表和结果段落的一体化工作流。

更适合对外传播的一句话版本：

> Inkly Cat：把科研数据变成可发表结果。

## 产品边界

不建议将 Inkly Cat 定义为：

- 万能 AI 助手
- 通用办公工作台
- 面向所有行业的数据分析平台

建议坚定定义为：

- AI-native scientific analysis workspace
- 面向科研与专业分析场景的结果交付工具
- 连接数据分析、统计检验、图表生成、结果写作、证据追溯的闭环产品

## 当前可用能力

### 学术研究与写作类

- `peer-review`：结构化论文或基金评审，基于清单评估方法学、统计有效性和报告规范
- `scientific-writing`：科学论文写作，遵循 IMRAD 结构，支持多种引用格式
- `literature-review`：系统性文献综述，支持多数据库检索和结构化合成
- `citation-management`：文献管理，搜索 Google Scholar / PubMed，生成 BibTeX

### 数据分析与可视化类

- `data-analysis-report`：数据分析报告，生成图表、表格和结构化发现
- `scientific-visualization`：出版级图表制作，支持多面板布局和期刊格式
- `plotly-figure-json`：生成可编辑的 Plotly 图表 JSON，适合前端集成

### 统计与假设类

- `statistical-analysis`：统计分析指导，帮助选择检验方法、假设检验和 APA 格式报告
- `hypothesis-generation`：结构化假设生成，从观察数据提出可检验假设
- `scientific-brainstorming`：开放性研究头脑风暴，探索跨学科联系

### 批判性思维与方法类

- `scientific-critical-thinking`：评估科学主张和证据质量，识别偏差和混杂因素
- `methodology-diagram-generator`：生成出版级方法学图、架构图和工作流程图

## 为什么这个方向成立

科研用户当前的真实问题不是缺少单点工具，而是流程碎片化：

- 数据在 Excel、CSV、R、Python、Prism 之间切换
- 文献在 PubMed、Google Scholar、Zotero 之间切换
- 图表在 Prism、AI 工具、Illustrator 之间切换
- 图注、结果段落、审稿回复依赖手工整理
- 统计方法和图表参数难以复核与回溯

因此，Inkly Cat 的竞争力应该来自于闭环，而不是功能堆叠。

## 对标产品判断

### Manus

Manus 更接近通用执行型 agent，卖点是任务广度和跨工具执行能力。

Inkly Cat 不应该在“什么都能做”上竞争，而应该在“专业结果更严谨、更可信”上竞争。

### Genspark

Genspark 更像 all-in-one AI workspace / AI employee，强调超级工作台、文档、表格、幻灯片和工作流整合。

Inkly Cat 不应该在“办公全家桶”上竞争，而应该在“科研分析结果产线”上竞争。

### Julius

Julius 的核心心智是聊天式数据分析。

Inkly Cat 需要继续往前走一层：不仅分析数据，还要生成期刊级图表、图注、结果段落，并提供方法与结论的可追溯性。

### GraphPad Prism

GraphPad Prism 占据的是科研统计 + 科研图表 + 生命科学用户心智。

Inkly Cat 不应该模仿成传统统计软件，而应该补上 Prism 没有做好的部分：

- AI 驱动分析路径建议
- 自动生成图注和结果写作
- 文献与证据闭环
- 团队协作和模板化输出
- 图表资产的持续可编辑性

## 最值得打的细分人群

建议先只打一个主战场，再逐步扩展。

### 第一优先级

生命科学、医学、心理学、药理学、公共卫生研究者

原因：

- 统计检验与图表要求高
- 发表导向明确
- 对可追溯和规范表达有真实需求
- 已经习惯为 Prism、SPSS、EndNote、Zotero 等专业工具付费

### 第二优先级

高校实验室中的硕博生、博后、RA、课题组骨干

原因：

- 高频用户
- 痛点集中
- 有较强传播性
- 更容易形成校园和实验室内扩散

### 第三优先级

CRO、科研服务公司、小型医药研发团队

原因：

- 客单价更高
- 对模板、审计、协作、安全的需求更强
- 更适合后续团队版和私有化

### 不建议优先切入的人群

- 通用办公用户
- 通用 BI 用户
- 泛互联网运营分析用户
- 泛行业“AI 员工”需求

这些方向会把你拉入 Manus、Genspark、Julius 这类产品的正面竞争。

## 核心差异化

### 1. 分析、出图、写作闭环

不是只做一段，而是连接：

- 原始数据
- 统计建议
- 图表生成
- 图注
- 结果段落
- 方法说明
- 证据引用

### 2. 统计与图表同时可审计

用户不只是得到一个答案，还能看到：

- 为什么推荐这个检验
- 检验的假设条件是什么
- 是否存在限制
- 图表对应的数据映射和参数是什么

### 3. 图表是可编辑资产

图表不应只是一次性生成的图片，而应是可以持续修改、复用、共享的资产。

这部分未来可以成为明显护城河：

- Plotly JSON
- 图层系统
- 样式系统
- 期刊 preset
- 模板复用

### 4. 面向发表，而不是面向演示

这一点需要成为品牌认知的核心。

Inkly Cat 不是为了做更好看的汇报图，而是为了帮助用户产出能进论文、基金、补充材料和审稿回复的结果。

## 12 个月发展路径

### 阶段一：0 到 3 个月

目标：做出一个非常清晰的价值楔子。

建议主线：

`上传表格 -> 自动建议分析 -> 生成图表 -> 输出图注和结果段落 -> 导出高质量图片和可编辑 JSON`

建议优先做好的图表类型：

- bar + error / CI
- box plot
- violin plot
- scatter
- correlation / regression
- survival
- ROC
- heatmap
- volcano
- dose-response

建议优先做好的专业能力：

- 自动推荐统计检验并解释原因
- 输出符合学术表达习惯的结果段落
- 提供 journal preset
- 提供图表导出和后续编辑能力

本阶段不建议投入太多资源去扩展泛用 agent 能力。

### 阶段二：3 到 6 个月

目标：把证据闭环接进来。

建议重点产品化以下能力：

- `literature-review`
- `citation-management`
- `peer-review`
- `scientific-writing`

形成新的产品闭环：

`找证据 -> 做分析 -> 出图 -> 写结果 -> 自查与审稿准备`

建议重点功能：

- 文献检索后自动形成引用资产
- 图表与结论可以关联文献证据
- reviewer mode 自动检查统计、图注和报告规范
- 结论可回链到原始数据与引用来源

### 阶段三：6 到 12 个月

目标：进入团队版与行业版。

建议重点能力：

- lab / project workspace
- 共享模板库
- 团队样式和统计 SOP
- 版本管理与批注
- 审批流
- 权限控制
- 审计日志
- 私有化或安全隔离方案

这一阶段更适合提升客单价。

## 宣传卖点

不建议把宣传重点放在“内置很多 skill”上。

建议把宣传重点放在用户能买单的结果上：

- 从表格到论文图，一步到位
- 自动推荐统计方法，并解释为什么
- 生成可编辑的期刊级图表
- 输出图注、结果段落和方法说明
- 每个统计结论都可追溯
- 面向发表，而不是面向演示

可对外使用的短句：

- 比通用 AI 更严谨，比统计软件更自动，比画图工具更懂发表
- 不是只会画图，是会给出正确分析路径
- 不只是生成结论，还保留结论背后的依据
- 图表可编辑，不锁死在一次生成里

## 定价建议

建议先用三层结构，不要过早复杂化。

### Free

适合体验：

- 限制项目数
- 限制导出分辨率或高级格式
- 提供基础图表和基础分析

### Pro

面向个人科研用户：

- 完整图表类型
- 完整统计建议
- 图注和结果段落生成
- journal preset
- 导出 JSON / SVG / PNG / PDF

### Team

面向实验室和科研服务团队：

- 共享模板
- 协作
- 权限
- 审计
- 团队知识库
- 私有数据控制

定价原则：

- 不要和通用 AI 聊天订阅直接对标
- 要和“节省科研产出时间的专业软件”对标
- 如果专业价值成立，Pro 可以明显高于通用 AI 月费

## 当前阶段最重要的产品承诺

如果只能抓一个最核心的承诺，建议是：

> 给我数据，我输出一套经得起追问的科研结果。

这句话成立以后，再扩展到文献、审稿、团队协作，逻辑才会顺。
